---
name: Lovable vite-tanstack-config dev server host/port on Replit
description: Why `bun/npm run dev` fails with EAFNOSUPPORT on Lovable-imported TanStack Start projects, and how to fix it.
---

Projects generated by Lovable using `@lovable.dev/vite-tanstack-config`'s `defineConfig` hardcode the
dev server to `host: "::"` (IPv6 any) and `port: 8080` unless a sandbox env var
(`LOVABLE_SANDBOX=1` or `DEV_SERVER__PROJECT_PATH`) is set. Outside that sandbox (e.g. a Replit
container), listening on `::` fails with `EAFNOSUPPORT: address family not supported :::8080`.

**Why:** the package's non-sandbox branch does `mergeConfig({server:{host:"::",port:8080}}, config)`,
where `config` includes whatever the project passed via `defineConfig({ vite: {...} })` — so a caller-supplied
`vite.server` override wins over the hardcoded default, but only if you fill it in yourself.

**How to apply:** in `vite.config.ts`, pass `vite: { server: { host: "0.0.0.0", port: 5000, strictPort: true, allowedHosts: true } }`
into the `defineConfig` call from `@lovable.dev/vite-tanstack-config`. Then configure the Replit workflow as
`bun run dev` (or `npm run dev`) with `waitForPort: 5000`, `outputType: "webview"`.
